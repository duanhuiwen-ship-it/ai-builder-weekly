# AI Builder 周报生成

自动抓取多个平台的 AI Builder 动态，生成面向产品经理的结构化周报。

## 快速开始

1. 确认需要抓取的数据源（使用默认配置或修改 `scripts/sources.json`）
2. 运行抓取脚本: `python scripts/fetch_content.py`
3. 运行报告生成脚本: `python scripts/generate_report.py`
4. 报告输出到当前目录: `AI_Builder_Weekly_Report_YYYY-MM-DD.md`

## 工作流程

```
Task Progress:
- [ ] 步骤 1: 确认时间范围（默认近 7 天）
- [ ] 步骤 2: 抓取 X/Twitter 账号动态
- [ ] 步骤 3: 抓取播客最新内容
- [ ] 步骤 4: 抓取博客最新文章
- [ ] 步骤 5: 整合数据生成周报
- [ ] 步骤 6: 验证报告完整性
```

### 步骤 1: 确认时间范围

默认抓取近 7 天的内容。如需自定义:

```bash
python scripts/fetch_content.py --start 2026-04-01 --end 2026-04-08
```

### 步骤 2-4: 抓取内容

使用并行子代理同时抓取不同平台:

- **X/Twitter**: 使用 WebSearch 搜索各账号近期推文
- **播客**: 搜索各播客频道的最新节目
- **博客**: 访问博客页面获取最新文章

抓取脚本会输出 JSON 格式的中间结果到 `scripts/output/fetched_data.json`，便于调试。

### 步骤 5: 生成周报

运行报告生成脚本:

```bash
python scripts/generate_report.py
```

报告结构:
```
# AI Builder 周报 | YYYY-MM-DD - YYYY-MM-DD

> **数据说明**: 标注哪些账号获取到了推文原文，哪些因反爬限制仅能获取摘要。

---

|------|------|----------|------------|
| 1 | ... | ... | ... |

### 整体 AI 发展趋势
（5 个方向，每个方向包含标题和 2-3 句综合摘要，基于本周多个来源的共同指向）
```

### 步骤 6: 验证

检查生成的报告:
- 每个平台至少有 2 条有效内容
- 所有 source 链接可访问
- 产品经理洞察具有 actionable 的建议

## 输出格式规范

### 周报头部

**数据说明**: 标注本周获取到推文原文的账号列表，以及因反爬限制仅能获取摘要的说明。

**本周速览 - Top 5**:
使用表格形式，精选本周最值得关注的 5 条内容：

```markdown
| 排名 | 来源 | 核心内容 | 一句话摘要 |
|------|------|----------|------------|
| 1 | Karpathy | LLM Knowledge Bases | 用 LLM 从"存储-检索"转向"生成-编译"范式 |
```

**本周速览 - 整体趋势**:
基于本周多个来源的共同指向，总结 5 个整体 AI 发展趋势方向。每个方向包含：
- 趋势标题
- 2-3 句综合摘要，引用本周具体来源的观点

### X/Twitter 动态条目格式

**优先使用推文原文**（如果能通过 WebSearch/WebFetch 获取到）:

```markdown
### 姓名 (@handle)
**推文原文**: [完整引用推文原文]
**source**: [推文链接]
**产品经理洞察**: [2-4 句对产品经理的启示，具体 actionable]
```

**如果无法获取原文**（X 平台反爬限制）:

```markdown
### 姓名 (@handle)
**输出内容**: [基于搜索引擎摘要的 1-3 句摘要]
**source**: [账号主页链接]
**产品经理洞察**: [2-4 句对产品经理的启示]
```

### 播客/博客动态格式

```markdown
### 来源名称
**输出内容**: [1-3 句摘要，包含核心观点]
**source**: [链接]
**产品经理洞察**: [2-4 句对产品经理的启示，具体 actionable]
```

## 周报质量标准

1. **覆盖度**: 至少覆盖 3 个平台，每个平台 2+ 条内容
2. **时效性**: 仅包含指定时间范围内的内容
3. **原文优先**: 优先获取推文原文，X/Twitter 因反爬限制无法获取原文时，使用搜索引擎摘要
4. **数据说明**: 在周报头部明确标注哪些账号获取到了原文，哪些仅获取摘要
5. **本周速览**: Top 5 表格和整体趋势总结必须基于本周实际内容，不可泛泛而谈
6. **洞察力**: 产品经理洞察必须具体、可操作，避免泛泛而谈
7. **准确性**: source 链接必须可访问且与内容匹配
8. **语言**: 使用中文，保持专业但易读的语气

## 抓取策略

### X/Twitter 原文获取方法

1. **直接搜索**: 使用 `WebSearch` 搜索 `"@handle keyphrase" site:x.com` 格式
2. **第三方站点**: 通过 `WebFetch` 访问 nitter/fixupx/bestblogs.dev 等镜像站点
3. **博客/论坛转载**: 搜索是否有第三方博客转载或引用了完整推文
4. **降级方案**: 如果以上都无法获取原文，使用搜索引擎摘要 + 明确标注

### 周报生成流程

1. 并行抓取 X/Twitter、播客、博客内容
2. 尝试获取推文原文（优先），否则使用摘要
3. 基于抓取内容生成 Top 5 表格和整体趋势总结
4. 按格式规范生成完整周报
5. 验证报告完整性

## 常见问题

**Q: 某些账号/平台没有新内容怎么办？**
A: 跳过该条目，仅报告有实际更新的来源。

**Q: 无法获取 X/Twitter 推文原文怎么办？**
A: 使用 WebSearch 搜索引擎摘要作为降级方案，并在周报头部的"数据说明"中明确标注哪些账号仅获取了摘要。

**Q: Top 5 如何选择？**
A: 基于以下标准：(1) 原创性和影响力；(2) 对产品经理的 actionable 价值；(3) 多个来源的共同指向。

**Q: 如何调整周报的语气和深度？**
A: 修改 `scripts/generate_report.py` 中的 prompt 模板。

**Q: 抓取失败如何处理？**
A: 脚本会输出错误日志到 `scripts/output/errors.log`，检查后重试失败的源。
